# 陀螺匠 API 开发规范

> 版本：1.0.0
> 作者：陀螺匠团队
> 更新日期：2026-03-24

## 2.1 技术规范

### 2.1.1 PHP语言规范

#### PHP版本要求

- **最低版本**：PHP 8.0+
- **推荐版本**：PHP 8.1+

#### 编码标准

- 遵循 PSR-12 编码标准
- 使用 4 个空格缩进
- 代码行宽度不超过 120 个字符

#### 类型声明

- 强制使用参数类型声明
- 强制使用返回类型声明
- 鼓励使用联合类型（PHP 8.0+）

```php
// 正确示例
public function getUserById(int $userId): ?User
{
    // 方法实现
}

// 错误示例（避免）
public function getUserById($userId)
{
    // 方法实现
}
```

#### 错误处理

- 使用异常（Exception）而非错误码
- 创建自定义异常类处理特定错误
- 记录异常日志便于排查问题

```php
// 正确示例
throw new UserNotFoundException("用户不存在，ID: {$userId}");

// 错误示例（避免）
return ['code' => 404, 'message' => '用户不存在'];
```

### 2.1.2 Laravel框架规范

#### 路由规范

- 使用路由属性（Route Attributes）定义路由
- API 路由统一添加 `api/` 前缀
- 使用资源路由简化 CRUD 操作

```php
// 正确示例
use App\Http\Controllers\UserController;

Route::apiResource('users', UserController::class);

// 错误示例（避免）
Route::get('/users', [UserController::class, 'index']);
Route::post('/users', [UserController::class, 'store']);
```

#### 控制器规范

- 控制器继承 AuthController
- 方法命名遵循 RESTful 规范
- 保持控制器方法简洁，只处理请求和响应

```php
// 正确示例
class UserController extends AuthController
{
    public function index(UserRequest $request): JsonResponse
    {
        $users = $this->userService->getUsers($request->validated());
        return $this->success($users);
    }
}
```

#### 模型规范

- 模型继承 BaseModel
- 使用 Eloquent ORM 进行数据库操作
- 定义表名、主键、fillable 等属性

```php
// 正确示例
class User extends Model
{
    protected $table = 'eb_user';
    protected $primaryKey = 'id';
    protected $fillable = ['username', 'email', 'phone'];
}
```

### 2.1.3 数据库规范

#### ORM使用

- 优先使用 Eloquent ORM
- 复杂查询使用 Query Builder
- 避免在控制器中直接写 SQL

```php
// 正确示例（使用 Eloquent）
$users = User::where('status', 1)
    ->orderBy('created_at', 'desc')
    ->paginate(20);

// 正确示例（使用 Query Builder）
$users = DB::table('users')
    ->where('status', 1)
    ->orderBy('created_at', 'desc')
    ->paginate(20);

// 错误示例（避免）
$users = DB::select("SELECT * FROM users WHERE status = 1");
```

#### 事务处理

- 关键操作使用数据库事务
- 合理控制事务范围
- 处理事务异常情况

```php
// 正确示例
DB::transaction(function () {
    $order = Order::create([...]);
    $this->orderService->updateStock($order);
    $this->notificationService->sendOrderNotification($order);
});
```

#### 索引规范

- 为高频查询字段创建索引
- 避免创建过多索引影响写入性能
- 使用复合索引优化多条件查询

```sql
-- 正确示例
ALTER TABLE eb_user ADD INDEX idx_status_created (status, created_at);

-- 错误示例（避免）- 为每个字段单独创建索引
ALTER TABLE eb_user ADD INDEX idx_status (status);
ALTER TABLE eb_user ADD INDEX idx_created (created_at);
```

#### 安全规范

- 使用参数绑定防止 SQL 注入
- 避免使用字符串拼接构建 SQL
- 对用户输入进行严格验证

```php
// 正确示例
$users = User::where('username', $username)->get();

// 错误示例（避免）
$users = DB::select("SELECT * FROM users WHERE username = '$username'");
```

### 2.1.4 性能优化

#### 查询优化

- 避免 N+1 查询，使用预加载（with）
- 使用分页处理大数据集
- 只查询需要的字段

```php
// 正确示例
$users = User::with(['profile', 'roles'])
    ->select(['id', 'username', 'email'])
    ->paginate(20);

// 错误示例（避免）
$users = User::all();
foreach ($users as $user) {
    $user->profile; // N+1 查询
}
```

#### 循环优化

- 避免在循环中执行数据库操作
- 使用批量操作替代循环操作
- 合理使用缓存减少数据库查询

```php
// 错误示例（避免）
foreach ($userIds as $userId) {
    $user = User::find($userId);
    // 处理逻辑
}

// 正确示例
$users = User::whereIn('id', $userIds)->get();
foreach ($users as $user) {
    // 处理逻辑
}
```

#### 缓存使用

- 合理使用缓存减少数据库查询
- 设置合理的缓存过期时间
- 及时清理失效缓存

```php
// 正确示例
$config = Cache::remember('system_config', 3600, function () {
    return SystemConfig::all();
});
```

## 2.2 代码风格

### 2.2.1 命名规范

#### 类名命名

- 使用 PascalCase 命名法
- 命名清晰表达类的功能
- 遵循统一的命名约定

```
UserController.php      ✓
userController.php     ✗
UsersController.php    ✗
```

#### 方法命名

- 使用 camelCase 命名法
- 方法名应表达动作和对象

```
getUserById()          ✓
GetUserById()          ✗
fetch_user_info()       ✗
```

#### 变量命名

- 使用 camelCase 命名法
- 命名应具有描述性
- 避免使用无意义的变量名

```
$userId                ✓
$user_name             ✗
$data                  ✗（应使用更具体的名称）
```

#### 常量命名

- 使用 SNAKE_CASE 命名法
- 全大写字母
- 使用有意义的单词组合

```
MAX_UPLOAD_SIZE        ✓
MAX_UPLOAD_SIZE_BYTES  ✓
maxUploadSize          ✗
```

#### 枚举类命名

- 使用 Enum 后缀
- 遵循 `*Enum.php` 命名规范

```
UserStatusEnum.php     ✓
UserStatus.php         ✗
```

### 2.2.2 代码格式

#### 缩进和空格

- 使用 4 个空格缩进
- 运算符前后添加空格
- 逗号后添加空格

```php
// 正确示例
$result = $a + $b;
$array = [1, 2, 3, 4];
function foo($arg1, $arg2) { }

// 错误示例
$result=$a+$b;
$array=[1,2,3,4];
function foo($arg1,$arg2){}
```

#### 空行使用

- 合理使用空行分隔代码块
- 方法之间使用空行
- 逻辑相关的代码行之间不使用空行

```php
// 正确示例
public function index()
{
    $users = $this->userService->getUsers();

    return response()->json(['data' => $users]);
}

public function show($id)
{
    $user = $this->userService->getUserById($id);

    return $this->success($user);
}
```

#### 注释规范

- 关键代码添加中文注释
- 注释应简洁明了
- 及时更新注释保持同步

```php
// 正确示例
// 根据用户ID获取用户信息
public function getUserById(int $id): ?User
{
    return User::find($id);
}

// 错误示例
// 获取用户
public function getUserById($id)
{
    return User::find($id);
}
```

#### 文档块注释

- 公共方法添加文档块
- 说明参数和返回值
- 使用 PHPDoc 格式

```php
/**
 * 获取用户信息
 *
 * @param int $id 用户ID
 * @return User|null 用户信息
 */
public function getUserById(int $id): ?User
{
    return User::find($id);
}
```

## 2.3 安全规范

### 2.3.1 认证授权

#### JWT认证

- 使用 JWT 进行 API 认证
- 设置合理的 token 过期时间
- 支持 token 刷新机制

```php
// 正确示例
$token = auth('api')->attempt(['username' => $username, 'password' => $password]);
return $this->success(['token' => $token]);
```

#### 权限控制

- 使用 Casbin 进行权限管理
- 定义清晰的权限节点
- 验证用户权限后再执行操作

```php
// 正确示例
if (!Casbin::enforce($user->role, 'users', 'create')) {
    abort(403, '无权限执行此操作');
}
```

#### 密码安全

- 使用 Laravel 哈希机制存储密码
- 设置密码强度要求
- 记录密码修改历史

```php
// 正确示例
$user->password = Hash::make($request->password);
```

### 2.3.2 输入验证

#### 请求验证

- 所有输入通过 Request 类验证
- 定义清晰的验证规则
- 返回友好的错误信息

```php
// 正确示例
class UserRequest extends ApiValidate
{
    public function rules()
    {
        return [
            'username' => 'required|string|max:50|unique:users',
            'email' => 'required|email|unique:users',
            'password' => 'required|string|min:6|confirmed',
        ];
    }
}
```

#### XSS防护

- 转义输出内容
- 使用 Blade 模板的转义功能
- 避免使用 raw HTML 输出

```php
// 正确示例
{{ $user->name }}

// 错误示例
{!! $user->name !!}  // 仅在确定安全时使用
```

#### CSRF防护

- API 认证不使用 Session 时可禁用 CSRF
- Web 表单启用 CSRF 保护
- 使用 token 验证请求来源

## 2.4 开发流程

### 2.4.1 版本控制

#### 分支管理

- 遵循 Git Flow 分支管理
- 主分支（main/master）保持稳定
- 开发分支（develop）集成新功能

```
main          - 生产环境代码
develop       - 开发分支
feature/*     - 功能分支
bugfix/*      - 修复分支
release/*     - 发布分支
```

#### 提交规范

- 提交信息清晰明了
- 使用中文描述提交内容
- 遵循统一的提交格式

```
feat: 添加用户注册功能
fix: 修复登录接口返回值错误
docs: 更新开发文档
style: 优化代码格式
refactor: 重构用户服务层
test: 添加单元测试
chore: 更新依赖包
```

### 2.4.2 代码审查

- 代码必须经过审查才能合并
- 审查重点：代码质量、安全性、性能
- 及时修复审查中发现的问题

### 2.4.3 测试要求

#### 单元测试

- 核心功能必须编写单元测试
- 测试覆盖业务逻辑
- 保持测试代码的可维护性

```php
// 正确示例
public function testUserCanLogin()
{
    $user = User::factory()->create([
        'username' => 'testuser',
        'password' => bcrypt('password'),
    ]);

    $response = $this->postJson('/api/auth/login', [
        'username' => 'testuser',
        'password' => 'password',
    ]);

    $response->assertStatus(200);
    $response->assertJsonStructure(['token']);
}
```

#### 测试数据

- 使用测试数据而非生产数据
- 使用 Factory 生成测试数据
- 保持测试数据的一致性

## 2.5 API开发规范

### 2.5.1 RESTful API 设计

#### 资源命名

- 使用复数名词表示资源
- 使用小写字母和下划线
- 避免使用动词

```
GET    /api/users          # 获取用户列表
GET    /api/users/{id}    # 获取单个用户
POST   /api/users          # 创建用户
PUT    /api/users/{id}    # 更新用户
DELETE /api/users/{id}    # 删除用户
```

#### HTTP 方法使用

- GET：查询资源
- POST：创建资源
- PUT：更新资源（完整更新）
- PATCH：更新资源（部分更新）
- DELETE：删除资源

### 2.5.2 响应格式

#### 成功响应

```json
{
    "code": 200,
    "msg": "success",
    "data": {
        "id": 1,
        "username": "admin",
        "email": "admin@example.com"
    }
}
```

#### 错误响应

```json
{
    "code": 400,
    "msg": "参数错误",
    "errors": {
        "username": "用户名不能为空"
    }
}
```

### 2.5.3 分页规范

#### 分页请求参数

- page：当前页码
- limit：每页数量
- sort：排序字段
- order：排序方式（asc/desc）

#### 分页响应格式

```json
{
    "code": 200,
    "msg": "success",
    "data": {
        "list": [...],
        "pagination": {
            "total": 100,
            "page": 1,
            "limit": 20,
            "total_pages": 5
        }
    }
}
```

### 2.5.4 状态码规范

| 状态码 | 说明 | 使用场景 |
|--------|------|----------|
| 200 | 成功 | 操作成功 |
| 201 | 已创建 | 资源创建成功 |
| 400 | 请求错误 | 参数错误、验证失败 |
| 401 | 未认证 | 未登录或token无效 |
| 403 | 无权限 | 无权限访问资源 |
| 404 | 未找到 | 资源不存在 |
| 500 | 服务器错误 | 服务器内部错误 |
