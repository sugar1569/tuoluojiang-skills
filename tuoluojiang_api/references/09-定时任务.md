## 9.1 定时任务概述

系统定时任务是指按照预定时间自动执行的脚本或命令，用于处理周期性的业务逻辑，如数据同步、报表生成、清理操作等。Laravel 框架提供了强大的任务调度系统，支持各种复杂的调度需求。

## 9.2 任务类型

### 9.2.1 核心任务类型

- **命令任务**: 执行 Artisan 命令
- **闭包任务**: 执行闭包函数
- **类任务**: 执行自定义任务类
- **队列任务**: 将任务推送到队列异步执行

### 9.2.2 执行频率

- **每分钟**: `everyMinute()`
- **每小时**: `hourly()`
- **每天**: `daily()`
- **每周**: `weekly()`
- **每月**: `monthly()`
- **每年**: `yearly()`
- **自定义**: 使用 `cron()` 方法设置自定义 Cron 表达式

## 9.3 任务创建

### 9.3.1 命令任务

```php
// app/Console/Kernel.php
protected function schedule(Schedule $schedule)
{
    // 执行 Artisan 命令
    $schedule->command('inspire')->hourly();
}
```

### 9.3.2 闭包任务

```php
// app/Console/Kernel.php
protected function schedule(Schedule $schedule)
{
    // 执行闭包函数
    $schedule->call(function () {
        // 任务逻辑
        DB::table('recent_users')->delete();
    })->daily();
}
```

### 9.3.3 类任务

1. **创建任务类**

```php
// app/Console/Commands/CleanupCommand.php
namespace App\Console\Commands;

use Illuminate\Console\Command;

class CleanupCommand extends Command
{
    protected $signature = 'cleanup:users';
    protected $description = '清理无效用户';
    
    public function handle()
    {
        // 任务逻辑
        $this->info('清理完成');
    }
}
```

2. **注册任务**

```php
// app/Console/Kernel.php
protected $commands = [
    Commands\CleanupCommand::class,
];

protected function schedule(Schedule $schedule)
{
    $schedule->command('cleanup:users')->daily();
}
```

### 9.3.4 队列任务

```php
// app/Console/Kernel.php
protected function schedule(Schedule $schedule)
{
    // 将任务推送到队列
    $schedule->job(new ProcessReports)->dailyAt('02:00');
}
```

## 9.4 任务调度机制

### 9.4.1 Cron 表达式

- **格式**: `分 时 日 月 周`
- **示例**: 
  - `* * * * *` - 每分钟执行
  - `0 * * * *` - 每小时执行
  - `0 0 * * *` - 每天执行
  - `0 0 * * 0` - 每周执行
  - `0 0 1 * *` - 每月执行

### 9.4.2 调度方法

- **基本频率**: `everyMinute()`, `hourly()`, `daily()` 等
- **特定时间**: `dailyAt('13:00')`, `weeklyOn(1, '8:00')` 等
- **条件执行**: `when()`, `skip()`, `environments()` 等
- **输出处理**: `sendOutputTo()`, `emailOutputTo()`, `appendOutputTo()` 等

### 9.4.3 任务重叠

- **防止重叠**: `withoutOverlapping()`
- **设置超时**: `timeout(60)` (秒)
- **维护模式**: `evenInMaintenanceMode()`

## 9.5 任务配置

### 9.5.1 核心配置文件

- **调度配置**: `app/Console/Kernel.php` - 定义所有定时任务
- **Cron 配置**: 服务器的 `crontab` - 配置 Laravel 调度器的运行

### 9.5.2 服务器配置

1. **编辑 Crontab**

```bash
crontab -e
```

2. **添加调度器任务**

```
* * * * * cd /path-to-your-project && php artisan schedule:run >> /dev/null 2>&1
```

3. **验证配置**

```bash
crontab -l
```

## 9.6 任务监控

### 9.6.1 日志监控

- **任务日志**: `storage/logs/` 目录下的日志文件
- **自定义日志**: 在任务中使用 `Log`  facade 记录日志

### 9.6.2 输出监控

- **控制台输出**: 使用 `$this->info()`, `$this->error()` 等方法
- **文件输出**: `sendOutputTo('path/to/log.txt')`
- **邮件通知**: `emailOutputTo('admin@example.com')`

### 9.6.3 第三方监控

- **Laravel Horizon**: 监控队列任务
- **Laravel Telescope**: 监控应用请求和任务
- **外部服务**: 如 Laravel Envoyer, Cronitor 等

## 9.7 最佳实践

### 9.7.1 任务设计

- **单一职责**: 每个任务只负责一个具体功能
- **错误处理**: 完善的异常捕获和处理机制
- **幂等性**: 任务重复执行不应产生副作用
- **超时控制**: 设置合理的超时时间

### 9.7.2 性能优化

- **异步处理**: 耗时任务使用队列
- **批量处理**: 大数据操作使用批量处理
- **资源管理**: 合理使用内存和数据库连接
- **调度优化**: 避免任务集中执行导致系统负载过高

### 9.7.3 安全考虑

- **权限控制**: 确保任务执行的权限正确
- **输入验证**: 任务参数的验证和过滤
- **敏感信息**: 避免在任务中处理敏感数据

## 9.8 常见问题

### 9.8.1 任务不执行

- 检查服务器 Crontab 是否正确配置
- 检查 `schedule:run` 命令是否可执行
- 检查任务的调度表达式是否正确
- 检查任务的 `when()` 条件是否满足

### 9.8.2 任务执行失败

- 检查任务代码是否有语法错误
- 检查任务的依赖是否满足
- 检查任务的异常处理是否完善
- 检查服务器资源是否充足

### 9.8.3 任务执行时间过长

- 优化任务代码，减少执行时间
- 将大任务拆分为多个小任务
- 使用队列异步执行耗时操作
- 调整任务的调度时间，避开系统高峰期

## 9.9 高级功能

### 9.9.1 任务链

- **定义任务链**: `$schedule->job(new Task1)->then(new Task2)->then(new Task3)->daily();`
- **条件任务链**: 基于前一个任务的结果决定是否执行下一个任务

### 9.9.2 任务分组

- **任务分组**: 将相关任务组织在一起，便于管理
- **并行执行**: 使用 `->runInBackground()` 让任务在后台并行执行

### 9.9.3 动态调度

- **运行时调度**: 根据系统状态动态调整任务调度
- **环境感知**: 根据不同环境设置不同的任务调度

## 9.10 定时任务优势

1. **自动化**: 减少人工干预，提高系统可靠性
2. **灵活性**: 支持各种复杂的调度需求
3. **可监控**: 完善的监控和日志机制
4. **可扩展性**: 易于添加和管理新任务
5. **可靠性**: 防止任务重叠和超时控制

通过了解和使用系统定时任务，可以实现各种周期性的业务需求，提高系统的自动化水平和可靠性，减少人工维护成本。
