> 版本：1.0.0

## 3.1 数据库概述

### 3.1.1 数据库信息

- **数据库类型**: MySQL
- **数据库名**: tuoluojiang
- **表前缀**: eb_
- **字符集**: utf8mb4
- **排序规则**: utf8mb4_unicode_ci
- **连接方式**: PDO (PHP Data Objects)

### 3.1.2 数据库设计原则

- 表名使用小写字母和下划线命名
- 字段名使用小写字母和下划线命名
- 主键统一使用 bigint 类型自增
- 时间字段使用 timestamp 类型
- 状态字段使用 tinyint 类型
- 软删除使用 deleted_at 字段

## 3.3 表关系

### 3.3.1 核心关系图

```
eb_admin ────┐
            │
            ▼
    eb_admin_info

    eb_user ────┐
            │
            ▼

    eb_approve ────┐
            │
            ▼

    eb_attendance_group ────┐
            │
            ▼
    eb_attendance_clock_record

    eb_assess
```

### 3.3.2 详细关系说明

1. **管理员与管理员信息**: 一对一关系，每个管理员对应一条详细信息
2. **用户与聊天历史**: 一对多关系，一个用户可以有多个聊天历史
3. **聊天历史与聊天记录**: 一对多关系，一个聊天历史可以有多个聊天记录
4. **聊天应用与聊天模型**: 多对一关系，多个聊天应用可以使用同一个聊天模型
5. **聊天应用与聊天历史**: 一对多关系，一个聊天应用可以有多个聊天历史
6. **审批与审批申请**: 一对多关系，一个审批流程可以有多个申请
7. **审批与审批表单**: 一对多关系，一个审批流程可以有多个表单字段
8. **考勤组与打卡记录**: 一对多关系，一个考勤组可以有多个打卡记录

## 3.4 数据库优化建议

### 3.4.1 索引优化

- **管理员表**: 为 account、name、phone 字段添加索引
- **用户表**: 为 username、email、phone 字段添加唯一索引
- **聊天历史表**: 为 user_id、chat_application_id 字段添加索引
- **聊天记录表**: 为 chat_history_id 字段添加索引
- **审批申请表**: 为 approve_id、status 字段添加索引
- **考勤打卡记录表**: 为 uid、group_id、created_at 字段添加索引
- **系统配置表**: 为 name 字段添加唯一索引

### 3.4.2 查询优化

- **避免全表扫描**: 使用 WHERE 子句和索引
- **合理使用 JOIN**: 只查询必要的字段
- **使用 LIMIT**: 分页查询时使用 LIMIT 限制结果集大小
- **避免 SELECT ***: 只选择必要的字段
- **使用覆盖索引**: 对于频繁查询的字段组合，创建覆盖索引

### 3.4.3 存储优化

- **分区表**: 对于打卡记录、聊天记录等大数据表，考虑使用分区
- **定期清理**: 定期清理过期的聊天记录、日志和打卡记录
- **使用缓存**: 对于频繁查询的数据，使用 Redis 缓存
- **数据归档**: 对于历史数据，考虑归档到历史表

## 3.5 数据库维护

### 3.5.1 备份策略

- **定期备份**: 每日全量备份，每周增量备份
- **备份验证**: 定期验证备份文件的完整性
- **备份存储**: 将备份文件存储在安全的位置
- **灾难恢复**: 制定详细的灾难恢复计划

### 3.5.2 监控建议

- **查询性能**: 监控慢查询日志，优化慢查询
- **空间使用**: 监控数据库空间使用情况，及时清理
- **连接数**: 监控数据库连接数，避免连接泄漏
- **错误日志**: 定期查看数据库错误日志，及时处理异常
