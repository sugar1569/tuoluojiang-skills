# 陀螺匠 API 项目概述

> 版本：1.0.0  
> 作者：陀螺匠团队  
> 更新日期：2026-03-24

## 1.1 项目简介

陀螺匠（Gyro Craftsman Owen）是一款基于 Laravel 9.x 框架开发的企业级应用管理系统。该系统采用模块化设计和分层架构，实现了用户管理、公司管理、审批流程、工作管理、财务管理、消息通知等核心功能。系统支持 JWT 认证和 Casbin 权限管理，提供完整的 API 接口，便于与其他系统集成和二次开发。

## 1.2 项目定位

陀螺匠是一个功能完善的企业级应用，旨在为企业提供高效、便捷的管理工具。系统设计遵循模块化原则，各功能模块之间低耦合、高内聚，便于维护和扩展。项目采用 PHP 8.0+ 开发，充分利用 PHP 8.0+ 的新特性，如命名参数、联合类型、属性等，提高代码的可读性和可维护性。

## 1.3 主要特性

### 1.3.1 模块化设计

系统采用模块化架构，每个功能模块都独立封装，具有清晰的职责边界。这种设计使得系统易于维护和扩展，开发者可以方便地添加新模块或修改现有模块，而不影响其他模块的正常运行。

### 1.3.2 分层架构

系统采用三层架构设计，分为表示层、业务逻辑层和数据访问层。表示层负责处理 HTTP 请求和响应，业务逻辑层实现核心业务规则，数据访问层负责与数据库交互。这种分层设计提高了代码的组织性和可测试性。

### 1.3.3 完整的认证授权体系

系统采用 JWT（JSON Web Token）实现用户认证，支持 token 刷新和过期处理。同时使用 Casbin 进行基于角色的访问控制（RBAC），支持细粒度的权限管理，确保系统的安全性。

### 1.3.4 异步任务处理

系统使用 Laravel Queue 处理异步任务，包括邮件发送、数据导出、消息推送等耗时操作。这种设计提高了系统的响应速度和用户体验。

### 1.3.5 缓存机制

系统支持 Redis 缓存，包括数据缓存、视图缓存、配置缓存等。通过合理的缓存策略，可以显著提高系统的性能和响应速度。

## 1.4 技术栈

### 1.4.1 后端技术

- **PHP**：8.0+（推荐 8.1+）
- **框架**：Laravel 9.x
- **数据库**：MySQL 5.7+ 或 MySQL 8.0+
- **缓存**：Redis 4.0+
- **队列**：Laravel Queue with Redis

### 1.4.2 认证授权

- **认证**：JWT（tymon/jwt-auth）
- **授权**：Casbin（lauthz/laravel-casbin）

### 1.4.3 扩展组件

- **Swoole**：LaravelS 用于提升性能
- **ORM**：Eloquent ORM
- **验证**：Laravel Validation

## 1.5 应用场景

陀螺匠适用于各类企业的内部管理系统，包括但不限于以下场景：

- 企业内部管理系统
- 项目管理系统
- 审批流程系统
- 考勤管理系统
- 绩效考核系统
- 客户关系管理系统
