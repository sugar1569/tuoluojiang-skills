## 10.1 中间件概述

中间件是 HTTP 请求处理流程中的过滤器，用于在请求到达控制器之前或响应返回客户端之前执行特定的操作。Laravel 框架提供了强大的中间件支持，系统基于此实现了完整的认证授权、日志记录、数据过滤等功能。

## 10.3 核心认证中间件

### 10.3.1 AuthAdmin - 后台管理员认证

**文件位置**: `app/Http/Middleware/AuthAdmin.php`

**功能说明**: 后台管理员身份认证中间件，验证管理员登录状态并处理 Token 刷新。

**主要功能**:
- 验证管理员 JWT Token 是否有效
- 自动刷新即将过期的 Token（剩余时间小于 10 小时）
- 将管理员信息绑定到请求对象

**使用方式**:

```php
#[Middleware(['auth.admin'])]
class UserController extends AuthController
{
    // 控制器方法
}
```

**配置位置**: 通常在路由属性中配置或 Kernel.php 中注册。

### 10.3.2 AuthUser - 前台用户认证

**文件位置**: `app/Http/Middleware/AuthUser.php`

**功能说明**: 前台用户身份认证中间件，用于验证前台用户的登录状态。

**主要功能**:
- 验证用户 Token 是否有效
- 加载用户信息到请求对象
- 处理 Token 过期和刷新逻辑

### 10.3.3 AuthEnterprise - 企业认证

**文件位置**: `app/Http/Middleware/AuthEnterprise.php`

**功能说明**: 企业认证中间件，用于验证企业用户并加载企业信息。

**主要功能**:
- 加载企业信息到请求对象
- 设置企业标识（entId、isEnt）
- 验证企业状态

**使用方式**:

```php
#[Middleware(['ent.auth'])]
class CompanyController extends AuthController
{
    // 控制器方法
}
```

### 10.3.4 AuthCrud - CRUD操作认证

**文件位置**: `app/Http/Middleware/AuthCrud.php`

**功能说明**: CRUD 操作权限认证中间件，用于验证用户对 CRUD 实体的操作权限。

**主要功能**:
- 验证 CRUD 操作权限（read、create、update、delete、transfer、share）
- 检查路由参数中的实体名称
- 验证 UUID 标识

**HTTP 方法对应权限**:

| HTTP 方法 | 权限类型 | 说明 |
|-----------|---------|------|
| GET | read | 读取权限 |
| POST | create | 创建权限 |
| PUT | update | 更新权限 |
| DELETE | delete | 删除权限 |
| TRANSFER | transfer | 转移权限 |
| SHARE | share | 分享权限 |

### 10.3.5 AuthOpenApi - 开放API认证

**文件位置**: `app/Http/Middleware/AuthOpenApi.php`

**功能说明**: 开放 API 认证中间件，用于第三方应用的 API 访问认证。

### 10.3.6 WebOfficeAuthUser - WebOffice用户认证

**文件位置**: `app/Http/Middleware/WebOfficeAuthUser.php`

**功能说明**: WebOffice 应用的用户认证中间件。

### 10.4.2 CheckRuleCompany - 企业权限检查

**文件位置**: `app/Http/Middleware/CheckRuleCompany.php`

**功能说明**: 检查企业用户是否有特定权限。

### 10.4.3 CheckEnterprise - 企业检查

**文件位置**: `app/Http/Middleware/CheckEnterprise.php`

**功能说明**: 检查企业状态和企业配置。

### 10.4.4 CheckCrudInfo - CRUD信息检查

**文件位置**: `app/Http/Middleware/CheckCrudInfo.php`

**功能说明**: 检查 CRUD 操作的相关信息是否完整。

### 10.4.5 CheckVersion - 版本检查

**文件位置**: `app/Http/Middleware/CheckVersion.php`

**功能说明**: 检查客户端版本是否满足要求。

## 10.5 日志中间件

### 10.5.1 LogEnterprise - 企业日志

**文件位置**: `app/Http/Middleware/LogEnterprise.php`

**功能说明**: 记录企业用户的操作日志。

**主要功能**:
- 记录请求信息（用户、时间、操作类型等）

- 支持异步写入日志

**使用方式**:

```php
#[Middleware(['auth.admin', 'ent.auth', 'ent.log'])]

{
    // 控制器方法
}
```

## 10.6 数据处理中间件

### 10.6.1 LangUage - 语言处理

**文件位置**: `app/Http/Middleware/LangUage.php`

**功能说明**: 处理多语言切换，根据请求头或参数设置应用语言。

**主要功能**:
- 读取语言设置（Accept-Language 或 lang 参数）
- 设置应用语言环境
- 支持语言切换

### 10.6.2 FilterStrings - 字符串过滤

**文件位置**: `app/Http/Middleware/FilterStrings.php`

**功能说明**: 过滤请求中的敏感字符串，防止 XSS 攻击。

### 10.6.3 TrimStrings - 字符串去空格

**文件位置**: `app/Http/Middleware/TrimStrings.php`

**功能说明**: 自动去除请求参数前后的空白字符。

**主要功能**:
- 去除 GET 参数的空白
- 去除 POST 参数的空白（除密码字段）
- 处理嵌套数组参数

### 10.6.4 ConvertEmptyStringsToNull - 空字符串转NULL

**文件位置**: `app/Http/Middleware/ConvertEmptyStringsToNull.php`

**功能说明**: 将空的字符串参数转换为 NULL 值。

## 10.7 安全中间件

### 10.7.1 EncryptCookies - Cookie加密

**文件位置**: `app/Http/Middleware/EncryptCookies.php`

**功能说明**: 加密和解密 Cookie 数据，防止 Cookie 篡改。

### 10.7.2 VerifyCsrfToken - CSRF令牌验证

**文件位置**: `app/Http/Middleware/VerifyCsrfToken.php`

**功能说明**: 验证 CSRF 令牌，防止跨站请求伪造攻击。

**使用场景**: 主要用于 Web 表单提交，不适用于 API 请求（API 使用 Token 认证）。

### 10.7.3 RedirectIfAuthenticated - 认证重定向

**文件位置**: `app/Http/Middleware/RedirectIfAuthenticated.php`

**功能说明**: 如果用户已认证，重定向到指定页面。

### 10.7.4 PreventRequestsDuringMaintenance - 维护模式拦截

**文件位置**: `app/Http/Middleware/PreventRequestsDuringMaintenance.php`

**功能说明**: 在维护模式下拦截所有请求，返回维护提示页面。

## 10.8 基础设施中间件

### 10.8.1 TrustProxies - 信任代理

**文件位置**: `app/Http/Middleware/TrustProxies.php`

**功能说明**: 配置信任的代理服务器，获取真实客户端 IP。

**配置示例**:

```php
protected $proxies = '*';  // 信任所有代理
protected $headers = Request::HEADER_X_FORWARDED_FOR;
```

### 10.8.2 TrustHosts - 信任主机

**文件位置**: `app/Http/Middleware/TrustHosts.php`

**功能说明**: 配置信任的主机域名，防止 DNS 重新绑定攻击。

## 10.9 中间件使用指南

### 10.9.1 在路由属性中使用

```php
#[Middleware(['auth.admin', 'ent.auth', 'ent.log'])]
class UserController extends AuthController
{
    #[Get('list', '用户列表')]
    public function index()
    {
        // 控制器方法
    }
}
```

### 10.9.2 在路由文件中使用

```php
Route::middleware(['auth.admin', 'ent.auth'])->group(function () {
    Route::get('/users', [UserController::class, 'index']);
    Route::post('/users', [UserController::class, 'store']);
});
```

### 10.9.3 中间件参数传递

```php
#[Middleware(['throttle:60,1'])]
class UserController extends AuthController
{
    // 60分钟内最多60次请求
}
```

## 10.10 中间件注册配置

### 10.10.1 HTTP Kernel 配置

在 `app/Http/Kernel.php` 中注册全局中间件：

```php
protected $middleware = [
    // 全局中间件
    \App\Http\Middleware\TrustProxies::class,
    \App\Http\Middleware\PreventRequestsDuringMaintenance::class,
    \App\Http\Middleware\TrimStrings::class,
    \App\Http\Middleware\ConvertEmptyStringsToNull::class,
];

protected $middlewareGroups = [
    'web' => [
        // Web 路由组中间件
    ],
    'api' => [
        // API 路由组中间件
    ],
];

protected $routeMiddleware = [
    // 路由中间件别名
    'auth.admin' => \App\Http\Middleware\AuthAdmin::class,
    'ent.auth' => \App\Http\Middleware\AuthEnterprise::class,
    'ent.log' => \App\Http\Middleware\LogEnterprise::class,
];
```

### 10.10.2 自定义中间件注册

```php
// 在 Kernel.php 中注册
protected $routeMiddleware = [
    'custom.middleware' => \App\Http\Middleware\CustomMiddleware::class,
];
```

## 10.11 中间件开发规范

### 10.11.1 创建自定义中间件

```php
<?php

namespace App\Http\Middleware;

use Closure;
use Illuminate\Http\Request;

class CustomMiddleware
{
    /**
     * 处理传入请求
     *
     * @param Request $request
     * @param Closure $next
     * @return mixed
     */
    public function handle(Request $request, Closure $next): mixed
    {
        // 前置处理逻辑
        
        $response = $next($request);
        
        // 后置处理逻辑
        
        return $response;
    }
}
```

### 10.11.2 实现 ApiMiddlewareInterface

对于需要使用项目中间件特性的中间件：

```php
<?php

namespace App\Http\Middleware;

use crmeb\interfaces\ApiMiddlewareInterface;
use crmeb\traits\MiddlewareTrait;
use Illuminate\Http\Request;

class CustomMiddleware implements ApiMiddlewareInterface
{
    use MiddlewareTrait;
    
    public function before(Request $request)
    {
        // 前置处理
    }
}
```

## 10.12 中间件使用示例

### 10.12.1 API 路由中间件配置

```php
#[Prefix('ent/user')]
#[Middleware(['auth.admin', 'ent.auth', 'ent.log'])]
class UserController extends AuthController
{
    /**
     * 用户列表
     */
    #[Get('list', '用户列表')]
    public function index()
    {
        // 需要认证：auth.admin
        // 需要企业认证：ent.auth  
        // 记录操作日志：ent.log
    }
    
    /**
     * 创建用户
     */
    #[Post('save', '创建用户')]
    #[Middleware(['auth.admin', 'ent.auth', 'ent.log', 'permission:user.create'])]
    public function store(UserRequest $request)
    {
        // 增加权限检查：permission:user.create
    }
}
```

### 10.12.2 频率限制中间件

```php
#[Middleware(['auth.admin', 'throttle:60,1'])]
class UserController extends AuthController
{
    // 60分钟内最多60次请求
}
```

## 10.13 中间件执行顺序

中间件按照注册顺序依次执行：

1. **全局中间件**: 在所有请求中执行
2. **路由组中间件**: 按路由组配置执行
3. **路由中间件**: 按单个路由配置执行
4. **控制器方法中间件**: 按控制器方法配置执行

## 10.14 中间件常见问题

### 10.14.1 中间件不生效

- 检查中间件是否正确注册
- 检查中间件别名是否正确
- 检查路由配置是否正确

### 10.14.2 中间件执行顺序错误

- 检查中间件注册顺序
- 调整中间件在 Kernel.php 中的位置

### 10.14.3 Token 刷新失败

- 检查 JWT 配置是否正确
- 检查 Token 是否已过期
- 检查刷新逻辑是否正确
