# 构建部署

## 1. 构建流程

### 1.1 安装依赖

```bash
# 安装依赖
npm install

# 使用 npm 镜像（推荐）
npm install --registry=https://registry.npmmirror.com
```

### 1.2 开发环境构建

```bash
# 启动开发服务器
npm run dev

# 访问地址
# http://localhost:8080
```

### 1.3 生产环境构建

```bash
# 构建生产版本
npm run build:prod

# 构建并发布
npm run build
```

## 2. 部署方式

### 2.1 静态文件部署

1. **Nginx 部署**

   ```nginx
   server {
     listen 80;
     server_name example.com;
     
     location / {
       root /path/to/dist;
       index index.html;
       try_files $uri $uri/ /index.html;
     }
   }
   ```

2. **Apache 部署**

   ```apache
   <VirtualHost *:80>
     ServerName example.com
     DocumentRoot /path/to/dist
     
     <Directory /path/to/dist>
       AllowOverride All
       Require all granted
     </Directory>
   </VirtualHost>
   ```

3. **IIS 部署**

   - 将 dist 目录复制到 IIS 网站根目录
   - 配置默认文档为 index.html
   - 配置 URL 重写规则，确保所有请求都指向 index.html

### 2.2 Docker 部署

1. **使用项目提供的 Docker 配置**

   ```bash
   # 构建 Docker 镜像
   docker build -t tuoluojiang:latest .
   
   # 运行容器
   docker run -d -p 8080:80 --name tuoluojiang tuoluojiang:latest
   ```

2. **使用 Docker Compose**

   ```yaml
   version: '3'
   services:
     tuoluojiang:
       image: tuoluojiang:latest
       ports:
         - "8080:80"
       volumes:
         - ./dist:/usr/share/nginx/html
   ```

### 2.3 云服务部署

1. **阿里云**

### 4.4 自动化部署

- **CI/CD**：使用 Jenkins、GitHub Actions 等实现自动化部署
- **版本控制**：使用 Git 管理代码版本
- **部署脚本**：编写自动化部署脚本
- **回滚策略**：实现快速回滚机制

## 5. 常见部署问题

### 5.1 路径问题

**问题**：部署后页面刷新出现 404 错误

**解决方案**：
- 配置服务器将所有请求重定向到 index.html
- 使用 hash 模式路由

### 5.2 资源加载问题

**问题**：静态资源加载失败

**解决方案**：
- 检查 `publicPath` 配置
- 确保资源路径正确
- 检查 CDN 配置

### 5.3 API 接口问题

**问题**：API 接口调用失败

**解决方案**：
- 检查 API 基础 URL 配置
- 检查 CORS 配置
- 检查网络连接

### 5.4 性能问题

**问题**：页面加载缓慢

**解决方案**：
- 优化代码分割
- 压缩静态资源
- 使用 CDN 加速
- 优化图片资源
