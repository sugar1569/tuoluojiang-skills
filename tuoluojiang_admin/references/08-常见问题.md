# 常见问题

## 1. 依赖安装问题

### 1.1 依赖安装失败

**问题描述**：运行 `npm install` 时出现安装失败的情况。

**解决方案**：
- 检查网络连接是否正常
- 清理 npm 缓存：`npm cache clean --force`
- 使用 npm 镜像：`npm install --registry=https://registry.npmmirror.com`
- 检查 Node.js 版本是否符合要求（>= 8.9）
- 检查 package.json 文件是否正确

### 1.2 依赖版本冲突

**问题描述**：安装依赖时出现版本冲突的警告或错误。

**解决方案**：
- 查看错误信息，确定冲突的依赖
- 使用 `npm ls` 命令查看依赖树
- 手动指定冲突依赖的版本
- 使用 `npm dedupe` 命令整理依赖

## 2. 构建问题

### 2.1 构建失败

**问题描述**：运行 `npm run build` 或 `npm run build:prod` 时构建失败。

**解决方案**：
- 检查代码语法错误
- 检查依赖版本冲突
- 检查文件权限
- 检查构建配置
- 查看详细的错误信息

### 2.2 构建速度慢

**问题描述**：构建过程耗时较长。

**解决方案**：
- 增加构建内存：`export NODE_OPTIONS=--max_old_space_size=4096`
- 使用增量构建
- 优化代码分割
- 减少不必要的依赖

## 3. 运行时问题

### 3.1 页面空白

**问题描述**：访问应用时页面显示空白。

**解决方案**：
- 检查浏览器控制台错误信息
- 检查网络请求是否正常
- 检查 API 接口连接
- 检查路由配置
- 检查权限配置

### 3.2 资源加载失败

**问题描述**：页面加载时资源（如图片、CSS、JS）加载失败。

**解决方案**：
- 检查资源路径是否正确
- 检查服务器配置
- 检查 CDN 配置
- 检查网络连接

### 3.3 API 接口调用失败

**问题描述**：API 接口调用返回错误或超时。

**解决方案**：
- 检查 API 基础 URL 配置
- 检查网络连接
- 检查后端服务是否正常
- 检查 CORS 配置
- 检查认证信息

### 3.4 控制台错误

**问题描述**：浏览器控制台出现错误信息。

**解决方案**：
- 查看详细的错误信息
- 定位错误位置
- 检查代码逻辑
- 检查数据格式
- 检查组件状态

## 4. 性能问题

### 4.1 页面加载缓慢

**问题描述**：页面加载时间过长。

**解决方案**：
- 组件懒加载
- 图片懒加载
- 代码分割
- 缓存策略
- 使用 CDN 加速
- 优化资源大小

### 4.2 运行时卡顿

**问题描述**：页面操作时出现卡顿现象。

**解决方案**：
- 减少 DOM 操作
- 使用虚拟列表
- 优化计算属性
- 避免频繁重渲染
- 使用 requestAnimationFrame

### 4.3 内存泄漏

**问题描述**：应用运行一段时间后内存占用过高。

**解决方案**：
- 检查事件监听器是否正确清理
- 检查定时器是否正确清除
- 检查组件销毁时的清理工作
- 使用 Chrome DevTools 分析内存使用情况

## 5. 兼容性问题

### 5.1 浏览器兼容性

**问题描述**：在某些浏览器中应用显示异常。

**解决方案**：
- 检查浏览器版本
- 使用 Babel 转译
- 使用 polyfill
- 测试主流浏览器
- 避免使用浏览器特定的 API

### 5.2 移动设备兼容性

**问题描述**：在移动设备上显示或操作异常。

**解决方案**：
- 响应式设计
- 触摸事件处理
- 移动端优化
- 测试不同设备

## 6. 安全问题

### 6.1 XSS 攻击

**问题描述**：存在跨站脚本攻击风险。

**解决方案**：
- 输入验证
- 输出编码
- Content-Security-Policy 配置
- 使用 Vue 的 v-html 指令时要谨慎

### 6.2 CSRF 攻击

**问题描述**：存在跨站请求伪造风险。

**解决方案**：
- CSRF 令牌验证
- 检查 Referer 头
- 使用 SameSite Cookie

### 6.3 敏感信息泄露

**问题描述**：敏感信息可能被泄露。

**解决方案**：
- 不在前端存储敏感信息
- 使用 HTTPS
- 合理设置 API 权限
- 避免在控制台输出敏感信息

## 8. 部署问题

### 8.1 路径问题

**问题描述**：部署后页面刷新出现 404 错误。

**解决方案**：
- 配置服务器将所有请求重定向到 index.html
- 使用 hash 模式路由
- 检查路由配置

### 8.2 环境配置

**问题描述**：不同环境的配置问题。

**解决方案**：
- 使用环境变量
- 配置不同环境的 .env 文件
- 检查 API 基础 URL 配置

### 8.3 容器部署

**问题描述**：Docker 容器部署相关问题。

**解决方案**：
- 检查 Dockerfile 配置
- 检查 Docker Compose 配置
- 检查容器网络配置
- 检查容器权限
