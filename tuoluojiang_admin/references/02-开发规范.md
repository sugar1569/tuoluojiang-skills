# 开发规范

## 1. 代码规范

### 1.1 ESLint 配置

项目使用 ESLint 进行代码检查，配置文件为 `.eslintrc.js`。

```bash
# 运行代码检查
npm run lint

# 自动修复代码问题
npm run lint -- --fix
```

### 1.2 Prettier 配置

项目使用 Prettier 进行代码格式化，配置文件为 `.prettierrc`。

```bash
# 自动格式化代码
npm run prettier
```

### 1.3 Vue 风格指南

遵循 Vue 官方风格指南，主要包括：

- 组件命名使用 PascalCase
- 方法命名使用 camelCase
- 变量命名使用 camelCase
- 常量命名使用 SNAKE_CASE
- 文件名使用 kebab-case

## 3. 命名规范

### 3.1 组件命名

- 组件名：PascalCase（如 `UserList.vue`）
- 文件名：kebab-case（如 `user-list.vue`）
- 组件内部方法：camelCase（如 `handleSubmit`）
- 组件内部变量：camelCase（如 `userName`）

### 3.2 API 命名

- API 文件：kebab-case（如 `user-management.js`）
- API 方法：camelCase（如 `getUserList`）
- API 路径：kebab-case（如 `/api/user/list`）

### 3.3 变量命名

- 普通变量：camelCase（如 `userName`）
- 常量：SNAKE_CASE（如 `API_BASE_URL`）
- 状态变量：camelCase（如 `isLoading`）
- 数组变量：复数形式（如 `users`）
- 布尔变量：前缀 `is` 或 `has`（如 `isActive`、`hasPermission`）

### 3.4 函数命名

- 普通函数：camelCase（如 `getUserInfo`）
- 事件处理函数：前缀 `handle`（如 `handleClick`）
- 生命周期函数：按照 Vue 规范（如 `mounted`、`beforeDestroy`）
- 计算属性：camelCase（如 `fullName`）

## 4. 代码风格

### 4.1 缩进

使用 2 个空格进行缩进，不使用制表符。

### 4.2 分号

语句结束使用分号。

### 4.3 引号

- 字符串使用单引号
- HTML 属性使用双引号
- 模板字符串使用反引号

### 4.4 括号

- 函数声明：`function name() {}`
- 条件语句：`if (condition) {}`
- 循环语句：`for (let i = 0; i < length; i++) {}`

### 4.5 空行

- 函数之间使用空行
- 逻辑块之间使用空行
- 文件末尾保留一个空行

## 5. 组件开发规范

### 5.1 组件结构

```vue
<template>
  <div class="component-name">
    <!-- 组件内容 -->
  </div>
</template>

<script>
export default {
  name: 'ComponentName',
  props: {
    // 属性定义
  },
  data() {
    return {
      // 数据定义
    }
  },
  computed: {
    // 计算属性
  },
  methods: {
    // 方法定义
  },
  mounted() {
    // 生命周期钩子
  }
}
</script>

<style scoped>
.component-name {
  /* 样式定义 */
}
</style>
```

### 5.2 Props 定义

- 明确指定类型
- 提供默认值
- 添加描述

```javascript
props: {
  title: {
    type: String,
    default: '标题',
    required: true
  },
  data: {
    type: Array,
    default: () => []
  }
}
```

### 5.3 事件处理

- 使用 `$emit` 触发事件
- 事件名使用 kebab-case
- 传递必要的参数

```javascript
// 触发事件
this.$emit('update', data)

// 监听事件
<component @update="handleUpdate" />
```

### 5.4 样式规范

- 使用 scoped 样式
- 类名使用 kebab-case
- 样式顺序：布局 > 盒模型 > 视觉 > 其他

```css
.component-name {
  /* 布局 */
  display: flex;
  justify-content: center;
  align-items: center;
  
  /* 盒模型 */
  width: 100%;
  height: 100px;
  padding: 16px;
  margin: 8px;
  
  /* 视觉 */
  background-color: #fff;
  border: 1px solid #e4e7ed;
  border-radius: 4px;
  
  /* 其他 */
  transition: all 0.3s;
}
```

## 6. API 开发规范

### 6.1 API 封装

使用 axios 封装 API 接口，统一处理请求和响应。

```javascript
import request from '@/api/request'

const userApi = {
  getList(params) {
    return request({
      url: '/api/user/list',
      method: 'get',
      params
    })
  }
}

export default userApi
```

### 6.2 请求参数

- GET 请求：使用 params
- POST/PUT 请求：使用 data
- 路径参数：使用模板字符串

### 6.3 响应处理

统一处理响应格式，包括成功和失败的情况。

```javascript
// request.js
import axios from 'axios'

const request = axios.create({
  baseURL: process.env.VUE_APP_API_BASE_URL,
  timeout: 10000
})

// 响应拦截器
request.interceptors.response.use(
  response => {
    const res = response.data
    if (res.code === 200) {
      return res
    } else {
      // 错误处理
      return Promise.reject(new Error(res.message || 'Error'))
    }
  },
  error => {
    // 错误处理
    return Promise.reject(error)
  }
)

export default request
```
