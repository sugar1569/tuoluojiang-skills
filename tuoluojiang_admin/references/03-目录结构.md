# 目录结构

## 1. 项目根目录

```
template/
├── patches/         # 补丁文件
├── public/          # 静态资源
├── src/             # 源代码
├── .eslintignore    # ESLint 忽略配置
├── .eslintrc.js     # ESLint 配置
├── README.md        # 项目说明
├── babel.config.js  # Babel 配置
├── docker-compose.yml # Docker 配置
├── index.js         # 入口文件
├── jest.config.js   # Jest 配置
├── jsconfig.json    # JavaScript 配置
├── package.json     # 项目配置
└── postcss.config.js # PostCSS 配置
```

## 2. 详细目录说明

### 2.1 patches/ 目录

存放项目补丁文件，用于修复依赖包的问题。

```
patches/
├── element-ui+2.15.14.patch     # Element UI 补丁
└── pdfjs-dist+2.0.943.patch     # PDF.js 补丁
```

### 2.2 public/ 目录

存放静态资源文件，这些文件会被直接复制到构建输出目录。

```
public/
├── image/           # 图片资源
│   └── logo.png     # 项目 logo
├── luckysheet/      # LuckySheet 插件
│   ├── assets/      # 插件资源
│   ├── css/         # 插件样式
│   ├── demoData/    # 示例数据
│   ├── fonts/       # 字体文件
│   ├── plugins/     # 插件扩展
│   ├── luckysheet.umd.js     # 核心文件
│   └── luckysheet.umd.js.map # 源码映射
├── README.md        # 说明文件
├── favicon.ico      # 网站图标
├── index.html       # HTML 模板
└── manifest.json    # PWA 配置
```

## 3. 目录结构设计原则

### 3.1 模块化

- 按功能模块划分目录
- 每个模块职责清晰
- 模块间低耦合高内聚

### 3.2 可扩展性

- 目录结构支持新增功能
- 易于添加新模块
- 支持插件化开发

### 3.3 可维护性

- 目录结构清晰易懂
- 代码组织合理
- 便于定位和修改

## 4. 目录使用建议

### 4.1 新增组件

- 按功能分类放置在对应目录
- 遵循命名规范
- 添加必要的文档

### 4.2 新增 API

- 按功能模块创建 API 文件
- 统一使用 request 封装
- 遵循 RESTful 规范

### 4.3 新增资源

- 按类型分类放置在对应目录
- 优化资源大小
- 考虑资源加载性能

## 5. 最佳实践

### 5.1 目录命名

- 使用小写字母和连字符
- 避免使用特殊字符
- 命名清晰明了

### 5.2 文件组织

- 相关文件放在同一目录
- 保持目录层级合理
- 避免过深的目录结构

### 5.3 代码组织

- 按功能组织代码
- 使用模块化开发
- 遵循单一职责原则
